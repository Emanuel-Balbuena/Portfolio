title: "Technical Evolution: An Architectural Journey Through My Projects"
date: "2026-03-27T13:00"
summary: "Reflections on my software development journey, from academic desktop apps to complex IoT architectures with voice integrations."
---

When I started programming, around 2022, I began my journey with a software called `PawLink`. I developed it in `Java` using `SceneBuilder`. The program consisted of a desktop application designed to facilitate the connection between veterinarians, shelters, and local users. It had a series of interesting features; however, at that early stage I still lacked deep technical knowledge and, most importantly, I considered it purely an academic project.

The result of that first iteration did not satisfy me. I felt that `SceneBuilder` limited my possibilities and did not give me total control over the interface. Therefore, as soon as I got a passing grade, I decided to refactor the entire program and migrate it from `Java` to `JavaScript` (`JS`). I carried out this process with the assistance of an `AI` which, back then, was still quite premature. This foundational project provided me with the technical knowledge necessary to scale into more complex systems and remains one of my favorite learning experiences.

Later on, I developed as professionally as possible a ticket manager called `NetworkLink` (built in `JS`). This system already integrated a small but structurally solid database, and consumed a `PHP` developed `API`. Despite being a relatively simple software, it allowed me to understand that I was barely scratching the surface of web development. Unfortunately, I lost the source code because back then I still didn't implement version control with `GitHub`. However, I don't consider it a grave loss, as today I possess the technical maturity to replicate it with a vastly superior architecture and without complications.

Today, one of my most ambitious and complete systems is `PowerLink`, a project I've been working on for an entire year. Its development has been an ongoing masterclass: I've integrated cloud services like `Supabase` and `Vercel`, and I've descended to the hardware level using pure electronics and `C++` for `Firmware` programming. I continue to constantly iterate over this ecosystem, and I can state that one of the most exciting technical milestones was developing the integration with `Alexa` services, which allows me to check telemetry in real-time and execute physical commands directly via voice.
